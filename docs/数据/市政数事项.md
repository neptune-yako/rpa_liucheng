# 市政数事项-交付与运营全景文档

> - **文档最后更新时间：** 2026-06-30

## 1. 系统概述与运行环境
- **1.1 项目背景**：政务管理系统中的事项目录包含大量的注册、修改、撤销、延伸、办事窗口编辑等工作。为了实现事项数据的自动化大批量同步，开发了此总控分发（Dispatcher）机器人流程。该主流程会自动轮询控制台与数据库推送表，自动清洗和过滤符合同步条件的事项记录，根据记录中的对象类型和操作类型，动态路由并分发给特定的业务子流程在浏览器前台执行对应操作，最终实现数据状态回写。
- **1.2 人机协同与触发机制**：本流程为**无人值守**模式。启动后流程进入常驻轮询循环状态，通过定时读取控制台变量与数据库触发执行。
- **1.3 技术选型与依赖组件**：
  - **工程版本**：V8.7.7
  - **主控流程**：`市政数事项.json`（ID：`hjMbS2Fp`）
  - **子流程依赖**：项目下包含 52 个依赖子流程（如 `主项目录注册.json`、`子项目录修订.json`、`更新数据库.json`、`机器人账号登录.json` 等）。
  - **核心组件**：RPA 浏览器组件 `Browser` (用于自动化登录与业务操作)、数据库连接模块 `mysql`、内置 Python 代码块。
  - **数据库**：MySQL 数据库（正式环境连接变量为 `conn`）。

## 3. 正常作业流程 (SOP)
- **3.1 步骤一：任务投递/触发阶段**：
  - 主流程启动并开始进行控制台变量 `是否新增任务数据` 的检查：
    - 若该变量值为 `0`，打印“0-不需要新增任务数据”，流程进入延迟后重新开始轮询。
    - 若该变量值为 `1`，主流程消费任务队列 **“亿讯二次录入推送数据”**，输出数据存入 `任务数据字典`，并利用 `Json.jsonLoads` 将其转化为字典。
- **3.2 步骤二：核心处理/交互阶段**：
  - **数据库推送记录检索**：
    - 调用子流程 `正式环境查询录入数据组` (`tksF7We7`) 查询 `rpa_item_province_push` 表中的待录入记录。
    - 遍历检索到的录入记录，针对每条记录校验状态 `status`：
      - `status == '9'`（异常）：关闭浏览器并终止。
      - `status == '2'`（正在同步）：说明已有其他实例在处理，关闭浏览器并终止。
      - `status == '1'` 或 `'N'`：已经同步，直接跳过并继续循环。
    - 过滤 `obj_type` 必须在 `['1','2','5','6','9','11']` 范围内，且 `oper_type` 必须在 `['0','1','2','3','4','6','7','8','9','10']` 范围内，不符则跳过。
  - **账号登录与分发路由 (Dispatcher)**：
    - 执行登录子流程，打开浏览器。根据记录的分类级别和同步省的标识判定，登录对应的账号。
    - 根据记录的 `obj_type`（对象）与 `oper_type`（操作）值组合，分发执行以下子流程模块：
      - **主项目录 (`obj_type == '1'`)**：
        - 注册 (`oper_type == '0'`) $\rightarrow$ 调用 `HPNFwfTu` (主项目录注册.json)
        - 修改 (`oper_type == '1'`) $\rightarrow$ 调用 `Rzj9csHV` (主项目录修改.json)
        - 撤销 (`oper_type == '2'`) $\rightarrow$ 调用 `H5qcqYu5` (主项目录撤销.json)
      - **子项目录 (`obj_type == '2'`)**：
        - 注册 (`oper_type == '0'`) $\rightarrow$ 调用 `Jy2DL6vH` (子项目录注册.json)
        - 修改 (`oper_type == '1'`) $\rightarrow$ 调用 `sgcnVNlG` (子项目录修订.json)
        - 撤销 (`oper_type == '2'`) $\rightarrow$ 调用 `mniQJQea` (子项目录撤销.json)
        - 取消纳入 (`oper_type == '5'`) $\rightarrow$ 调用 `uYxJnzfG` (子项目录取消纳入.json)
      - **事项压减 (`obj_type == '5'`)**：
        - 发起主项压减 (`oper_type == '0'`) $\rightarrow$ 调用 `dbGHfa9C` (发起主项事项压减.json)
        - 发起子项压减 (`oper_type == '1'`) $\rightarrow$ 调用 `Iv2RCc2W` (发起实施清单(子项)事项压减.json)
      - **实施清单 (`obj_type == '6'`)**：
        - 补全实施清单 (`oper_type == '0'`) $\rightarrow$ 调用 `jIX1Nlvr` (子项目录补全实施清单.json)
        - 修订实施清单 (`oper_type == '1'`) $\rightarrow$ 调用 `MULTTuq1` (子项目录修订实施清单.json)
        - 撤销实施清单 (`oper_type == '2'`) $\rightarrow$ 调用 `tdG7U12G` (子项目录撤销实施清单.json)
        - 编辑流程环节 (`oper_type == '8'`) $\rightarrow$ 调用 `ZEpUiayp` (实施清单编辑流程环节.json)
        - 删除流程环节 (`oper_type == '9'`) $\rightarrow$ 调用 `eNwsLYDO` (实施清单删除流程环节.json)
        - 修改标签 (`oper_type == '10'`) $\rightarrow$ 调用 `ID7O6fBJ` (实施清单修改标签.json)
      - **延伸基层 (`obj_type == '9'`)**：
        - 延伸基层 (`oper_type == '4'`) $\rightarrow$ 调用 `o2C3MFSY` (子项目录延伸基层.json)
        - 基层绑定 (`oper_type == '6'`) $\rightarrow$ 调用 `tzSa2zaA` (子项目录基层绑定.json)
        - 取消延伸 (`oper_type == '7'`) $\rightarrow$ 调用 `n8fTn8Ih` (子项目录取消延伸.json)
      - **办事窗口 (`obj_type == '11'`)**：
        - 新增窗口 (`oper_type == '0'`) $\rightarrow$ 调用 `0Tj6FC7D` (新增办事窗口信息.json)
        - 修改窗口 (`oper_type == '1'`) $\rightarrow$ 调用 `Mv4ssf7V` (修改办事窗口信息.json)
- **3.3 步骤三：结束与回填阶段**：
  - 分发子流程执行完毕后，主流程会调用子流程 `ohsC0p3a` (`更新数据库.json`) 将处理结果回写至 MySQL 数据库。
  - 在 `finally` 阶段，一律关闭数据库连接（`mysql.disconnectServer`），并计算循环耗时，若单次轮询时间小于 60 秒则自动执行 `sleep` 补足时间，防止接口请求过于频繁。
- **3.4 重入与并发规则说明**：
  - **重试性**：本流程为持久化数据同步，对报错中断的记录，可在库中将其状态修改回初始值以重新触发分发。
  - **并发性**：涉及 Chrome 浏览器的前台录入，建议单机单实例串行运行，避免并发切换账号导致浏览器假死。

## 4. 业务异常提示与人工干预指南
- **4.1 业务拦截异常处理**：
  - 当子业务流程遭遇数据校验失败等错误时，主流程的 `catch` 块会捕获异常，并使用 `Browser.browser_ele_screenshot` 截取当前浏览器报错画面（保存为 `截图`），随后调用 `ohsC0p3a` 标记记录状态为失败。运维人员需查看保存的 `截图` 并人工干预。
- **4.2 熔断/超时异常处理**：
  - 当查到同步状态 `status` 为 `2` (正在同步) 时，流程会熔断并跳出当前任务周期，以防止多机器人冲突。

## 6. 常见技术故障排查（FAQ）
- **6.1 网络与通信故障**：
  - **现象**：数据库连接不上。
  - **排查步骤**：检查数据库 IP `10.194.252.40` 端口 `8066` 是否正常开放，局域网策略是否有网络拦截。
- **6.2 组件与数据异常**：
  - **现象**：高频率切换登录账号导致政务平台响应慢或假死。
  - **方案**：主流程在账号登录切换前设置了延迟（147小节），保证操作有时间缓冲。出现假死后，`finally` 会关闭浏览器重新建立干净环境。
