## 1. 系统概述与运行环境
- **1.1 项目背景**：在政务事项数据大批量推送至省系统的过程中，为保障同步的逻辑顺序和下游机器人的高效分批处理，本流程作为后台队列生成器运行。它读取数据库中的事项同步推送流，清洗排除无效数据，随后利用 Python 的 Pandas 库将推送记录按“事项名称”与“组织机构”进行分组，将同一事项的分组记录进行时序排序打包，投递至任务队列中供下游消费机器人顺序处理。
- **1.2 人机协同与触发机制**：本流程为**无人值守**模式。通过定时循环轮询控制台控制变量 `是否新增任务数据` 进行触发。
- **1.3 技术选型与依赖组件**：
  - **工程版本**：V8.7.7
  - **主控流程**：`根据名称分组新增任务数据.json`（ID：`2NWw3vL8`）
  - **子模块依赖**：
    * 数组生成子流程：`生成数组.json`（ID：`3jhlazJo`）
    * Excel 写入子流程：`写入excel.json`（ID：`xYMUXGeM`）
  - **核心组件**：RPA 数据库组件 `mysql`、任务队列控制组件 `TaskDataItem`、内置 Python 代码块（调用 Pandas 与 Numpy 进行数据分组）。
  - **网络通信与环境**：双数据库连接（外部 MySQL 及本地局域网 MySQL）。

## 3. 正常作业流程 (SOP)
- **3.1 步骤一：任务投递/触发阶段**：
  - 主流程启动后，获取控制台变量 `是否新增任务数据`：
    - 若 `新增标识 == 0`，打印“不需要新增任务数据”，流程延迟后继续下一次循环。
    - 若 `新增标识 == 1`：
      - **防卡死检测**：计算当前时间与该标识上次更新时间的时间差。如果时间差超过 **600 分钟（10 小时）**，判定为卡死故障，强制把 `新增标识` 重置为 `0` 并更新回控制台以继续轮询。
      - 若无卡死，主流程连接数据库 `10.194.252.40:8066` 并执行查询：
        ```sql
        select * from rpa_item_province_push where status not in ('N','1') order by create_time,obj_type,oper_type
        ```
- **3.2 步骤二：核心处理/交互阶段**：
  - **子模块处理（数组清洗与生成）**：
    1. 调用子流程 `生成数组.json` (`3jhlazJo`)。
    2. 子流程连接本地数据库 `10.196.193.23:3306`，遍历记录检查对应的 `oper_user_name` 在 `用户账号对应表` 中的机构及 Cookie 会话状态。
    3. **排除规则**：排除无可用 Cookie 凭据的用户记录（如果满足目录级别 catalog_level == '4' 且 sync_province == '0' 的区级直推事项除外）。
    4. **类型过滤**：校验 `obj_type` 必须在 `['1','2','5','6','9','11']` 范围内，否则跳过。
    5. **名称匹配**：根据 `obj_type` 分别从事项源表（如 `r_rpa_catalog_oper`、`v_rpa_item_oper`、`v_rpa_extend_apply`、`v_rpa_window_info`）解析获取该记录在系统中的实际 `事项名称`。
    6. 将符合清洗规则的数据打包并返回 `待写入数组` 给主流程。
  - **Pandas 组装分发**：
    1. 主流程接收返回数组，转成 Pandas DataFrame，并执行以下分组逻辑：
       ```python
       df = pd.DataFrame(data[1:], columns=data[0])
       分组 = df.groupby(['事项名称','组织机构'], sort=False)
       ```
    2. 循环遍历各分组：
       - 对组内记录按 `'创建时间'、'操作对象'、'操作类型'` 升序排序。
       - **拦截过滤**：若该分组存在状态为 `'9'`（异常）的记录，或组内完全不存在 `'0'`（未同步）的记录，直接跳过并继续循环。
       - 将时间、状态等字段转换为字符串类型。
       - 调用 `TaskDataItem.addData` 写入队列 **“亿讯二次录入推送数据”**，写入数据负载为 `{'推送数据': 列表}`（列表内为该组排序后的所有事项修改/新增操作序列）。
- **3.3 步骤三：结束与回填阶段**：
  - 数据写入队列完成后，主流程将控制台 `是否新增任务数据` 的 `新增标识` 重置为 `0`，并断开数据库连接。
  * **特别说明**：调用子流程 `写入excel` (`xYMUXGeM`) 的第 58 步节点在主流程配置中已被**禁用**。因此，本流程正常运行时**不会**向本地 `D:\二次录入并发运行\二次录入.xlsx` 导出文件。
- **3.4 重入与并发规则说明**：
  - **重试性**：本流程为只读分析及队列生成器，中途报错终止后可重新执行。
  - **并发性**：属于后台计算流程，不进行任何浏览器界面操作，适合常驻在后台服务中单实例运行。

## 6. 常见技术故障排查（FAQ）
- **6.1 网络与通信故障**：
  - **现象**：提示无法连接到 `10.196.193.23` 数据库。
  - **排查步骤**：检查数据库实例运行状态、用户名密码是否过期，并使用 `telnet 10.196.193.23 3306` 进行连通性测试。
- **6.2 组件与数据异常**：
  - **现象**：运行至 Pandas 分组块报错，提示数据格式不支持或 Null 值异常。
  - **方案**：检查从 `生成数组` 传回的数组是否包含表头索引，或者推送表中是否存在脏数据破坏了结构。
