# 用户粤政易扫码-交付与运营全景文档

> - **文档最后更新时间：** 2026-06-30

## 1. 系统概述与运行环境
- **1.1 项目背景**：本流程为省系统扫码服务端的子流程模块，专门用于在前台浏览器上实现扫码界面的自动化引导与监控。它负责将目标系统登录页的粤政易二维码以 Base64 图片数据的形式进行截取，并通过 Socket 发送给客户端，同时通过循环检测网页元素判断用户的实际扫码登录结果。
- **1.2 人机协同与触发机制**：本流程由主服务端主流程（`省系统扫码登录服务端.json`）在调用子流程时触发。执行过程中需要在浏览器中操作页面跳转、截图、并阻塞轮询等待人工手机端“粤政易”APP 进行扫码。
- **1.3 技术选型与依赖组件**：
  - **流程名称**：用户粤政易扫码
  - **模块文件**：`用户粤政易扫码.json`（ID：`OgYcI2gg`）
  - **主要组件**：RPA 浏览器组件 `Browser`（控制跳转、点击、截图）、`Control` 控制流（循环控制、条件分支）

---

## 2. 运行前准备
- **2.1 系统与环境检查**：
  - 本子流程在运行时依赖于主流程传入的活动浏览器句柄（参数 `hWeb`）和网络套接字句柄（参数 `conn_socket`）。
- **2.2 软硬件就绪确认**：
  - 必须确保被调用的浏览器窗口处于最前且可被正常点击控制。
  - 需要外部网络与统一认证平台 `tyrz.gdgov.cn` 联通以正常显示二维码。
- **2.3 安全与凭据合规说明**：
  - 扫码动作全由人工手机端配合完成，子流程本身不保存或录入任何粤政易用户名或个人密码凭据。

---

## 3. 正常作业流程 (SOP)
- **3.1 步骤一：任务投递/触发阶段**：
  - 服务端主流程将其接收到的 `hWeb`、`conn_socket` 以及 `组织机构` 等参数传递并启动本流程。
- **3.2 步骤二：核心处理/交互阶段**：
  - **网页点击与跳转**：
    1. 查找并点击网页中的“已绑定，点击马上登录”链接（XPath：`//a[text()="已绑定，点击马上登录"]`）。
    2. 若由于网络等原因未发现按钮，则调用 `Browser.refresh` 刷新页面并通过 `clickElementByJs` 行使 JS 强制点击。
    3. 若判断“粤政易”文字标识（XPath：`//span[contains(text(),'粤政易')]`）未出现，则调用 `navigateURL` 强制跳转至统一身份认证登录地址。
  - **二维码截取与回传**：
    1. 定位 iframe 标签 `//iframe[contains(@src,'https://zwwx.gdzwfw.gov.cn/wwopen/sso/qrConnect')]` 内的二维码图片元素 `//img[contains(@class,'qrcode lightBorder')]`。
    2. 使用元素截图组件（`Browser.browser_ele_screenshot`）截取二维码，并将生成的 Base64 编码存入 `screenshot`。
    3. 通过传入的 `conn_socket` 句柄回传图像字节流给客户端：
       ```python
       conn_socket.sendall(str(screenshot).encode('UTF8'))
       ```
- **3.3 步骤三：结束与回填阶段**：
  - **扫码检测循环**：
    - 进入 `while True` 条件循环，每隔数秒检测以下页面元素的显示状态以决定登录结果：
      - **成功登录**：当页面出现包含主页“首页”字样的元素（XPath：`//span[text()='首页']`）时，设置输出参数 `首页 = True`，跳出循环返回。
      - **多账号选择**：当检测到选择账号界面（XPath：`//div[text()='选择要登录的省级系统账号']`）时，根据传入的 `组织机构` 变量定位对应的选择框并点击（XPath：`//span[text()='%s']/parent::td/preceding-sibling::td[3]/input` % 组织机构），随后点击提交按钮（XPath：`//button[contains(text(),'提交省级系统')]`）。
      - **扫码无效/未绑定**：当检测到登录失败提示框（XPath：`//*[@id='loginFailed']`）时，设置输出参数 `红叉 = True`，跳出循环返回。
- **3.4 重入与并发规则说明**：
  - 本流程是交互型子流程，其运行生命周期受浏览器会话和手机扫码超时的约束。如发生扫码超时或失败，需重新由主流程发起调用。

## 4. 业务异常提示与人工干预指南
- **4.1 业务拦截异常处理**：
  - 扫码后若检测到绑定关系异常（触发 `//*[@id='loginFailed']`），子流程返回 `红叉 = True`，提醒主控流程向外部平台发起业务失败上报（即“没有绑定省账号”）。
- **4.2 熔断/超时异常处理**：
  - 本子流程在等待用户扫码时，若在循环等待设定的时间（或超时次数）后页面仍未跳转到主页，将返回超时状态，需由外部主流程执行清理和重新触发。

## 5. 数据库设计与状态机
- 本子流程完全由前台浏览器视图驱动，不与 MySQL 数据库建立连接。它通过输出变量 `首页` 和 `红叉` 向主流程传递状态逻辑。

## 6. 常见技术故障排查（FAQ）
- **6.1 网络与通信故障**：
  - **现象**：网页停留于登录页，没有出现二维码，日志抛出元素未找到异常。
  - **排查步骤**：检查前台浏览器是否能打开 `zwwx.gdzwfw.gov.cn` 域名下的统一认证二维码，若网络受限或域名解析失败，请检查运行端机器的 DNS 或代理配置。
- **6.2 组件与数据异常**：
  - **现象**：多组织机构选择时报错。
  - **排查步骤**：检查传入的 `组织机构` 字符串是否与省系统页面上显示的文本（完全匹配）一致。如果 XPath 替换失败，说明页面组织机构文字存在差异，或对应的 input 框定位失效。

## 7. 运维服务单提交规范
- **7.1 日志获取路径与截取方法**：
  - 技术人员可通过查询运行端日志，过滤子流程特有的日志标识如“已发送端口消息给客户端”或相关的 XPath 点击异常。
- **7.2 故障提单要素**：
  - 提供出错的 `useraccount`、页面停留处的 Chrome 浏览器窗口截图、以及对应时间段的日志报错。
- **7.3 磁盘与缓存清理**：
  - 截图 `screenshot` 是在内存中完成 Base64 编码并以字节流形式通过网络直接发送的，无需在本地磁盘生成物理文件，因此本子流程不会占用本地磁盘空间。
